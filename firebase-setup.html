Update Firestore Security Rules

0/3

Open the Rules editor

Firestore Database → Rules tab
Open Rules Editor ↗

Replace the rules with the correct version

Select all existing text in the editor and replace with this (also saved as firestore.rules in your project folder):

// BizSoft Solutions — Firestore Security Rules v2
rules_version = '2';

service cloud.firestore {
  match /databases/{database}/documents {

    // leads — public CREATE (forms), auth READ/UPDATE/DELETE (console)
    match /leads/{leadId} {
      allow create: if
        request.resource.data.keys().hasAll(['name', 'phone', 'ticketId']) &&
        request.resource.data.name  is string && request.resource.data.name.size()  > 0 && request.resource.data.name.size()  < 120 &&
        request.resource.data.phone is string && request.resource.data.phone.size() > 4 && request.resource.data.phone.size() < 20 &&
        request.resource.data.ticketId is string &&
        request.resource.data.keys().size() <= 20;

      allow read, update, delete: if request.auth != null;
    }

    // plans — public READ (live pricing), auth write (console)
    match /plans/{planId} {
      allow read: if true;
      allow create, update, delete: if request.auth != null;
    }

    // everything else — deny
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
⚠ The old rules block the admin console from reading leads. You MUST update them or the console will show "⚠ API 403" when loading leads.

Publish the rules

Click the Publish button in the Firebase rules editor. Changes take effect within ~30 seconds.

✔ After publishing: website forms can still submit leads, the website shows live pricing, and the console can read/manage everything — but nobody else can.

Firebase Hosting (Deploy the Site)

0/3

Install Firebase CLI (one-time)

Open a terminal / command prompt on your PC and run:

npm install -g firebase-tools
Requires Node.js. Download from nodejs.org if not installed (LTS version).

Login and initialise hosting

In a terminal, cd into your project folder and run these commands one by one:

firebase login
firebase use bizsoftsolutions-8846b
firebase init hosting

When init asks questions, answer:

  • Public directory? → type . (dot — the current folder)
  • Single-page app? → No
  • Overwrite index.html? → No
A firebase.json and .firebaserc file will be created — that's normal, don't delete them.

Deploy the site

Every time you want to push updates live, run:

firebase deploy --only hosting

Firebase gives you a live URL like https://bizsoftsolutions-8846b.web.app. You can also connect your custom domain bizsoftsolutions.in:

Hosting → Add custom domain → enter bizsoftsolutions.in → follow DNS steps
Open Hosting ↗
✔ Firebase Hosting includes free SSL, CDN, and 10 GB bandwidth/month — plenty for this site.
